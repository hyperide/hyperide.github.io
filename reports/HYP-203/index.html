Architecture Decisions

ADR-1: Deterministic-first preview generation Click to expand
Problem
Preview file generation required AI API key and network calls, blocking offline / no-key users.
Options
1) Keep AI-only 2) AST-based with Babel 3) Regex scanner + string templates
Decision
Regex scanner + string templates (option 3). AI as fallback only.
Rationale
Regex is fast, Babel-free (no bundle cost for VS Code extension), and sufficient for detecting export patterns. Template strings produce deterministic, predictable output.
ADR-2: Unified AI client over direct SDK imports Click to expand
Problem
13 files imported Anthropic SDK directly. Adding OpenAI-compatible providers required changes everywhere.
Options
1) Adapter per file 2) Single callAI abstraction 3) Middleware/proxy layer
Decision
Single callAI/callAIStream abstraction with ResolvedAIConfig discriminated union.
Rationale
Minimal API surface. Provider routing centralized. Anthropic SDK for native features, fetch-based SSE for OpenAI-compatible.
ADR-3: FileIO abstraction for portability Click to expand
Problem
PreviewFileManager must work in both Node.js (server) and VS Code extension (virtual filesystem).
Options
1) Conditional imports 2) FileIO interface with DI 3) Abstract class
Decision
FileIO interface with dependency injection (readFile, writeFile, access).
Rationale
Simple 3-method interface. InMemoryFileIO for tests. NodeFileIO for server. Extension provides its own implementation.
