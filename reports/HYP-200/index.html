Product Changes

✨ What's New
  • Unified Diagnostic Panel
    Single panel for all diagnostic logs — server stdout/stderr, iframe console, compilation errors. Replaces the old DockerLogsViewer with a modern, filterable interface.
    New: DiagnosticLogsViewer.tsx, DiagnosticFilterBar.tsx, diagnosticStore.ts
  • Severity-Based Filtering
    Filter logs by severity level — Error, Warning, Info, Debug. Active severity counts shown in filter bar badges.
    New: useDiagnosticFilter.ts, DiagnosticFilterBar.tsx
  • Real-Time Search & Highlight
    Full-text search across log messages with real-time highlighting. Matches are highlighted inline without regex — safe from ReDoS.
    New: highlight.tsx, highlightSearch.test.tsx
  • Log Persistence to Postgres
    Diagnostic logs are batched and persisted to a Postgres table via a new API route. Logs survive page refresh and can be queried historically.
    New: diagnostic-logs.ts (route), diagnostic-logs.ts (schema), migration 0008
  • VS Code DiagnosticHub
    Central aggregator in the VS Code extension that collects logs from DevServerManager, PreviewPanel, and iframe, forwarding them to the webview logs panel.
    New: DiagnosticHub.ts, DiagnosticPersistenceService.ts, useDiagnosticSyncExt.ts
  • Iframe Console Capture
    Injected script intercepts console.log/warn/error in the preview iframe and forwards entries to the parent window via postMessage.
    New: iframe-console-capture.js
🚀 Improvements
  • Delta Broadcasting
    SSE now broadcasts only new log entries (delta) instead of the full buffer. Drastically reduces bandwidth and re-render cost at high log volume.
    Changed: useDiagnosticSync.ts
  • Exponential Backoff for SSE Reconnection
    SSE reconnection now uses exponential backoff (1s → 2s → 4s → ... → 30s max) instead of immediate retry, reducing server load during outages.
    Changed: useDiagnosticSync.ts
  • Semantic Color Tokens
    All severity colors now use semantic Tailwind tokens (bg-destructive, text-warning) instead of hardcoded values, ensuring consistent dark theme support.
    Changed: DiagnosticFilterBar.tsx
🔧 Fixes
  • Persistence Loop Prevention
    Fixed infinite loop where store.subscribe → persist → store update → subscribe chain would never terminate. Dual-ref guard breaks the cycle.
    Fixed: diagnosticStore.ts
  • Persistence Stuck After Clear
    After clearing logs, persistence refs were not reset, causing new logs to never be persisted. Now resets both refs on clear.
    Fixed: diagnosticStore.ts
  • XSS via Regex Injection
    Search input was passed directly to RegExp constructor. Replaced with safe String.includes() matching.
    Fixed: highlight.tsx
  • Missing Auth on Diagnostic Routes
    POST/GET /api/diagnostic-logs were accessible without authentication. Added inline authMiddleware + requireEditor.
    Fixed: diagnostic-logs.ts (route)
  • SSE Connection Leak
    EventSource was never closed on component unmount. Added cleanup in useEffect return.
    Fixed: useDiagnosticSync.ts
