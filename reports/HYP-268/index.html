Architecture: Click-to-Source Flow

Migration Obstacles

Issues that appeared specifically during the transition to fiber-based tracing — not reasons for the migration, but problems that had to be solved to make it work end-to-end.

P1 Turbopack strips leading slash from _debugSource.fileName

Turbopack emits /Users/alex/project/app/page.tsx as Users/alex/project/app/page.tsx — no leading /. goToCode() received a path VS Code couldn't resolve. Fixed in EditorBridge.ts: prepend / when path looks absolute but lacks the slash.

P1 Turbopack uses indexed source maps (sections format)

Turbopack bundles multiple source files into a single chunk and uses a top-level "sections" array instead of a flat "mappings" string. The resolver threw on this format. Fixed: detect sections format, iterate each section's offset to find the right one for the clicked line/column.

P1 SyncPositionService: _pendingSource consumed after early-return guard

The fiber click path sends a SourceLocation with the click event. But the service checked _getCurrentComponent() first — when no .tsx file was open, it returned early and dropped the source location silently. Click-to-source never worked in Next.js because of this. Fixed: consume _pendingSource before the active-component check. Regression test added.

P2 Source map resolver returned node_modules paths for React internals

When clicking elements close to React internal boundaries (e.g. Suspense boundaries), source map resolution sometimes returned a node_modules/react/ path. Navigation opened a library file instead of user code. Fixed: skip results pointing to node_modules or .yarn/cache, fall back to last valid user-land result.

P2 Async source map resolution: click lost if maps not yet fetched

Next.js Turbopack loads source maps lazily. On first click after page load, the source map request was in-flight when the click handler ran, returning null. Click was silently dropped. Fixed: retry loop in iframe-interaction.ts with 5-second timeout. Click is buffered until the map arrives.

P3 Next.js App Router reserved files broke preview entry point detection

Preview generator picked up layout.tsx, loading.tsx, error.tsx as candidate entry points — these are App Router convention files, not renderable standalone components. Fixed: exclude known App Router reserved filenames from entry point heuristic.
